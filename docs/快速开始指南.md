## 🚀 快速启动（3步）

### 步骤1：初始化数据库

```bash
# 1. 登录MySQL
mysql -u root -p

# 2. 创建数据库
CREATE DATABASE medical_platform CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 3. 退出MySQL
exit;

# 4. 导入表结构（如果有init.sql文件）
mysql -u root -p medical_platform < database/init.sql
```

### 步骤2：启动后端

```bash
# 1. 进入后端目录
cd backend

# 2. 编译项目（首次运行需要下载依赖，可能需要几分钟）
mvn clean install

# 3. 启动Spring Boot应用
mvn spring-boot:run

# 看到以下日志表示启动成功：
# Started MedicalPlatformApplication in X.XXX seconds
```

**后端启动在**：http://localhost:8080

**测试API**：http://localhost:8080/api/patients/list

### 步骤3：启动前端

```bash
# 1. 进入前端目录
cd frontend

# 2. 安装依赖（首次运行）
npm install

# 3. 启动开发服务器
npm run dev

# 看到以下信息表示启动成功：
# Local:   http://localhost:3000
```

**前端访问地址**：http://localhost:3000

## 📖 项目结构快速理解

### 后端结构（Java）

```
backend/
├── src/main/java/com/medical/
│   ├── controller/     # API接口层（接收HTTP请求）
│   ├── service/        # 业务逻辑层（处理业务）
│   ├── mapper/         # 数据访问层（操作数据库）
│   ├── entity/         # 实体类（对应数据库表）
│   └── config/         # 配置类（Redis、Security等）
└── src/main/resources/
    ├── mapper/         # MyBatis SQL映射文件
    └── application.yml  # 配置文件
```

### 前端结构（Vue 3）

```
frontend/
├── src/
│   ├── views/          # 页面组件
│   ├── components/     # 可复用组件
│   ├── api/            # API请求封装
│   └── router/         # 路由配置
└── package.json         # 依赖管理
```

## 🔍 理解数据流转（核心）

### 最简单的例子：查询患者列表

```
1. 用户在浏览器访问：http://localhost:3000/patients
   ↓
2. Vue前端发送HTTP请求：GET /api/patients/list
   ↓
3. Spring Boot Controller接收请求
   ↓
4. Controller调用Service
   ↓
5. Service调用Mapper查询数据库
   ↓
6. MySQL返回数据
   ↓
7. 数据原路返回：Mapper → Service → Controller → JSON响应
   ↓
8. 前端接收JSON数据，渲染到页面
```

### 代码对应关系

```
前端：frontend/src/views/PatientList.vue
  ↓ 调用API
API：frontend/src/api/patient.js
  ↓ HTTP请求
后端Controller：backend/.../controller/PatientController.java
  ↓ 调用方法
后端Service：backend/.../service/PatientService.java
  ↓ 调用方法
后端Mapper：backend/.../mapper/PatientMapper.java
  ↓ 执行SQL
数据库：MySQL patient_visit表
```

## 💡 新手常见问题

### Q1: 后端启动失败，提示端口被占用

**解决**：
```bash
# Windows查看端口占用
netstat -ano | findstr :8080

# 杀死进程（替换PID为实际进程ID）
taskkill /PID <PID> /F

# 或者修改application.yml中的端口
server:
  port: 8081  # 改为其他端口
```

### Q2: 数据库连接失败

**检查**：
1. MySQL是否启动
2. `application.yml`中的数据库配置是否正确
3. 数据库是否已创建

**修改配置**：
```yaml
# backend/src/main/resources/application.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/medical_platform
    username: root
    password: your_password  # 修改为你的密码
```

### Q3: 前端npm install失败

**解决**：
```bash
# 清除缓存
npm cache clean --force

# 使用淘宝镜像
npm config set registry https://registry.npmmirror.com

# 重新安装
npm install
```

### Q4: Redis连接失败

**解决**：
1. 确保Redis已启动
2. 检查`application.yml`中的Redis配置
3. 如果不需要Redis，可以先注释掉相关代码
