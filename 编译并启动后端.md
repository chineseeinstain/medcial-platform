# 手动编译并启动后端服务

## 问题原因
UserController是新创建的文件，需要重新编译才能被Spring Boot加载。

## 解决步骤（Windows PowerShell）

### 方法1：使用Maven命令（推荐）

1. **打开PowerShell，进入backend目录**：
   ```powershell
   cd D:\code\java\MDEICAL-PLATFORM\backend
   ```

2. **停止当前运行的后端服务**（如果正在运行）：
   - 在运行后端的终端按 `Ctrl+C` 停止
   - 或者查找占用8080端口的进程并结束：
     ```powershell
     netstat -ano | findstr :8080
     # 找到PID后，使用以下命令结束进程（替换YOUR_PID为实际PID）：
     taskkill /PID YOUR_PID /F
     ```

3. **清理并编译项目**：
   ```powershell
   mvn clean compile
   ```

4. **启动后端服务**：
   ```powershell
   mvn spring-boot:run
   ```

5. **等待启动完成**，看到以下信息说明启动成功：
   ```
   Started MedicalPlatformApplication
   Tomcat started on port(s): 8080
   ```

6. **检查UserController是否加载**：
   在启动日志中查找：
   ```
   UserController 初始化完成
   UserService 注入状态: ✅ 已注入
   ```

### 方法2：使用IDE（如果使用IntelliJ IDEA或Eclipse）

1. **停止当前运行的服务**

2. **重新编译项目**：
   - IntelliJ IDEA: `Build` -> `Rebuild Project`
   - Eclipse: `Project` -> `Clean...` -> 选择项目 -> `Clean`

3. **重新启动应用**：
   - IntelliJ IDEA: 右键 `MedicalPlatformApplication.java` -> `Run 'MedicalPlatformApplication'`
   - Eclipse: 右键项目 -> `Run As` -> `Spring Boot App`

## 验证修复

启动成功后，在浏览器访问：
```
http://localhost:8080/api/users/list
```

应该返回用户列表的JSON数据，而不是404错误。

## 如果仍然404

1. 检查后端日志中是否有错误信息
2. 确认UserController.java文件在正确的位置：`backend/src/main/java/com/medical/controller/UserController.java`
3. 检查编译后的class文件是否存在：`backend/target/classes/com/medical/controller/UserController.class`
