## 📚 后端技术栈

### 1. Java 8
**是什么**：编程语言  
**作用**：编写后端业务逻辑  
**关键点**：面向对象、跨平台、JVM运行

### 2. Spring Boot 2.7
**是什么**：Java框架，简化Spring开发  
**作用**：
- 自动配置，无需手动配置XML
- 内置Tomcat服务器
- 快速搭建RESTful API

**常用注解**：
- `@RestController`：标记控制器，返回JSON
- `@RequestMapping("/api/xxx")`：定义API路径
- `@Autowired`：自动注入依赖
- `@GetMapping`、`@PostMapping`：HTTP方法

**示例**：
```java
@RestController
@RequestMapping("/api/users")
public class UserController {
    @Autowired
    private UserService userService;
    
    @GetMapping("/list")
    public Result<List<User>> getUserList() {
        return Result.success(userService.getAllUsers());
    }
}
```

### 3. Spring Security
**是什么**：安全框架  
**作用**：
- 用户认证（登录验证）
- 权限控制（角色管理）
- JWT Token生成和验证

**核心概念**：
- **认证（Authentication）**：你是谁？→ 登录验证
- **授权（Authorization）**：你能做什么？→ 权限检查

### 4. MyBatis
**是什么**：数据库ORM框架  
**作用**：将Java对象和数据库表映射，执行SQL查询

**工作方式**：
1. 定义Mapper接口（Java接口）
2. 编写SQL映射文件（XML）
3. 调用接口方法执行SQL

**示例**：
```java
// Mapper接口
public interface UserMapper {
    List<User> selectAll();
}

// XML映射文件
<select id="selectAll" resultType="User">
    SELECT * FROM user
</select>
```

### 5. MySQL
**是什么**：关系型数据库  
**作用**：存储数据（表、行、列）

**基本操作**：
- `SELECT`：查询数据
- `INSERT`：插入数据
- `UPDATE`：更新数据
- `DELETE`：删除数据

**本项目使用**：存储患者、科室、医保等12张表

### 6. Redis
**是什么**：内存数据库（缓存）  
**作用**：
- 缓存热点数据，提升查询速度
- 存储Session、Token等临时数据

**为什么用**：MySQL查询慢，Redis查询快（内存vs磁盘）

### 7. Maven
**是什么**：项目管理和构建工具  
**作用**：
- 管理依赖（自动下载jar包）
- 编译、打包项目

**常用命令**：
- `mvn clean compile`：清理并编译
- `mvn spring-boot:run`：启动项目
- `mvn package`：打包成jar

### 8. JWT（JSON Web Token）
**是什么**：无状态认证方式  
**作用**：用户登录后生成Token，后续请求携带Token验证身份

**流程**：
1. 用户登录 → 后端验证用户名密码
2. 验证成功 → 生成JWT Token返回前端
3. 前端存储Token → 后续请求Header中携带Token
4. 后端验证Token → 确认用户身份

## 🎨 前端技术栈

### 1. Vue 3
**是什么**：前端框架  
**作用**：构建用户界面，数据驱动视图

**核心概念**：
- **响应式数据**：数据变化，页面自动更新
- **组件化**：将页面拆分成可复用组件
- **组合式API**：使用`setup()`函数组织代码

**示例**：
```vue
<template>
  <div>{{ message }}</div>
</template>

<script setup>
import { ref } from 'vue'
const message = ref('Hello Vue')
</script>
```

### 2. Vue Router
**是什么**：Vue的路由管理器  
**作用**：实现单页应用（SPA）的页面跳转

**核心概念**：
- **路由**：URL路径对应哪个页面组件
- **路由守卫**：页面跳转前的权限检查

**示例**：
```javascript
// 路由配置
{
  path: '/dashboard',
  component: Dashboard
}
```

### 3. Pinia
**是什么**：Vue的状态管理工具  
**作用**：管理全局共享数据（如用户信息）

**为什么用**：多个页面需要共享数据时，避免层层传递

### 4. Element Plus
**是什么**：Vue 3的UI组件库  
**作用**：提供现成的组件（按钮、表格、表单等）

**常用组件**：
- `<el-button>`：按钮
- `<el-table>`：表格
- `<el-form>`：表单
- `<el-card>`：卡片

### 5. ECharts
**是什么**：数据可视化图表库  
**作用**：绘制各种图表（折线图、柱状图、饼图等）

**使用场景**：展示统计数据、趋势分析

### 6. Axios
**是什么**：HTTP请求库  
**作用**：向后端API发送请求，获取数据

**示例**：
```javascript
// GET请求
axios.get('/api/users/list')
  .then(response => {
    console.log(response.data)
  })

// POST请求
axios.post('/api/auth/login', {
  username: 'admin',
  password: '123456'
})
```

### 7. Vite
**是什么**：前端构建工具  
**作用**：
- 开发时：快速启动开发服务器
- 生产时：打包优化代码

**对比**：比Webpack更快，启动速度提升10倍+

## 🏗️ 项目架构

### 前后端分离
```
前端（Vue 3）
  ↓ HTTP请求（JSON格式）
后端（Spring Boot）
  ↓ 返回JSON数据
前端（渲染页面）
```

### 后端三层架构
```
Controller层（控制器）
  ↓ 调用
Service层（业务逻辑）
  ↓ 调用
Mapper层（数据访问）
  ↓ SQL查询
MySQL数据库
```

**各层职责**：
- **Controller**：接收HTTP请求，返回JSON响应
- **Service**：处理业务逻辑（如：验证、计算）
- **Mapper**：执行SQL，操作数据库

## 🔑 核心概念速记

### RESTful API
**是什么**：API设计规范  
**规则**：
- GET：查询数据
- POST：创建数据
- PUT：更新数据
- DELETE：删除数据

**示例**：
- `GET /api/users/list` → 获取用户列表
- `POST /api/users` → 创建用户
- `PUT /api/users/1` → 更新用户1
- `DELETE /api/users/1` → 删除用户1

### JSON
**是什么**：数据格式（JavaScript对象表示法）  
**作用**：前后端数据交换的标准格式

**示例**：
```json
{
  "code": 200,
  "message": "成功",
  "data": {
    "id": 1,
    "username": "admin"
  }
}
```

### 跨域（CORS）
**是什么**：浏览器安全机制  
**问题**：前端（localhost:5173）访问后端（localhost:8080）被阻止  
**解决**：后端配置CORS，允许跨域请求

### 数据脱敏
**是什么**：保护隐私数据  
**做法**：不存储真实姓名、身份证号，使用脱敏后的ID

## 📝 面试常见问题

### Q1: 为什么用Spring Boot？
**答**：简化配置，快速开发，内置服务器，生态丰富

### Q2: MyBatis和JPA的区别？
**答**：
- MyBatis：写SQL，灵活，适合复杂查询
- JPA：不写SQL，简单，适合简单CRUD

### Q3: Vue 2和Vue 3的区别？
**答**：
- Vue 2：Options API（data、methods分开写）
- Vue 3：Composition API（setup函数，逻辑更集中）

### Q4: 为什么用Redis？
**答**：缓存热点数据，提升查询速度，减轻数据库压力

### Q5: JWT Token的优势？
**答**：无状态（不需要服务器存储Session），适合分布式系统
